Debouncing and throttling are techniques in JavaScript used to control how often a function is executed, especially in response to frequent events such as scrolling, resizing, or keypresses. They are particularly useful for improving performance and user experience.

Definition: Debouncing ensures that a function is executed only after a specified delay has elapsed since the last time it was invoked. It prevents a function from being called repeatedly in quick succession.

Here after clicking the on the increament button, button pressed will increase by 1 in every presse but for debouncing if we release the press for 8 ms than triggered count will increase by 1.

Use Case:

1- Search bar suggestions (wait for the user to stop typing before making a search query).

2- Resize event handling (trigger logic only after resizing is finished).

Definition: Throttling ensures that a function is executed at most once every specified interval, even if the event is triggered multiple times during that interval.

Use Case:

1- Scroll event handling (execute a function at regular intervals during scrolling).

2- Button click prevention to avoid rapid, repeated submissions.
